# Diplom

Сайт для интерактивной 3D-визуализации учебного оборудования.

## Что реализовано

- выбор специальности: ОИБ, ПД, ЗЕМ;
- каталог учебного оборудования для каждой специальности из JSON-базы;
- интерактивный просмотр 3D-модели в браузере;
- чистые серверные маршруты вида `/equipment/terrain-relief`;
- QR-код для открытия выбранной карточки оборудования на домене учреждения;
- серверный API для генерации QR-кода в формате base64;
- логирование открытий моделей по QR-коду в `logs/scan.log`;
- интерактивные аннотации и режим «Сетка» для 3D-макета.

## Локальный запуск на Linux

```bash
npm install
npm start
```

После запуска откройте:

```text
http://localhost:8080
```

Если открыть `index.html` двойным кликом как файл, интерфейс также загрузит
стили и локальную копию каталога. Но QR-коды и логирование работают только через
Node.js backend, поэтому для защиты и проверки используйте `npm start`.

Сервер Express слушает `0.0.0.0`, поэтому сайт можно открыть с других
устройств в локальной сети по адресу Linux-компьютера, например:

```text
http://192.168.1.25:8080
```

Если сайт открыт на самом сервере через `localhost` или `127.0.0.1`, backend
динамически определяет IPv4-адрес сетевого интерфейса Linux-компьютера и
подставляет его в QR-ссылку. Поэтому телефон студента получает адрес сервера в
локальной сети, а не собственный `127.0.0.1`.

Для дипломной пояснительной записки: «В системе реализован механизм
динамического определения IP-адреса сервера. При генерации QR-кода backend
проверяет адрес запроса и, если пользователь открыл сайт через loopback-адрес,
заменяет его на актуальный IPv4-адрес сетевого интерфейса сервера. Это
обеспечивает корректное открытие карточек оборудования с мобильных устройств в
локальной сети».

## Backend API

```text
GET /api/specialties
GET /api/equipment
GET /api/equipment/:equipmentId
GET /api/qr/:equipmentId
GET /equipment/:equipmentId
```

Список специальностей и оборудования хранится в `data/equipment.json`.
Добавление новой специальности или объекта выполняется добавлением новой записи
в этот файл, после чего интерфейс обновляется через API без правок HTML.

QR-ссылка ведет на `/equipment/:equipmentId?scan=1`. При открытии такой ссылки
сервер записывает событие просмотра в `logs/scan.log`.

## Размещение на сервере учебного заведения

Проект запускается как Node.js-приложение. Для размещения скопируйте файлы
проекта на сервер, установите зависимости и запустите процесс:

```bash
cd /var/www/equipment-3d
npm install --omit=dev
HOST=0.0.0.0 PORT=8080 npm start
```

Пример Nginx-конфигурации:

```nginx
server {
    listen 80;
    server_name equipment.college.local;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

Если сайт должен всегда генерировать QR-коды с публичным доменом, задайте:

```bash
PUBLIC_BASE_URL=https://equipment.college.ru npm start
```

Локальная CSS-3D-визуализация работает без интернета. Ссылки на GLB-модели в
`data/equipment.json` можно заменить на файлы учреждения, например
`models/camera.glb`.
